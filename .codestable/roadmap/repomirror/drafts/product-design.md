# RepoMirror 产品设计

## 1. 产品定位

`RepoMirror` 是一个 `Go + Wails` 的 Windows 桌面工具。

它的目标很明确：

- 管理两个固定 Git 仓库目录
- 展示当前同步方向下的差异文件
- 执行双向镜像同步
- 在目标仓库上继续做 `commit` / `push`

它不是通用 Git 客户端，也不是脚本启动器。

## 2. 界面设计原则

- 把主要空间留给差异文件列表
- 尽量减少无关说明的视觉占用
- 让用户一眼知道当前方向、差异数量、目标仓库是否干净
- 把 Git 提交/推送入口放在目标仓库状态旁边
- 不做大块日志区

## 3. 首版界面结构

界面分成三块：

### 3.1 顶部控制区

只保留高频操作：

- A 仓库路径
- B 仓库路径
- 选择目录
- 交换 A/B
- 保存
- `A → B`
- `B → A`
- `执行同步`
- `刷新`

### 3.2 中央主体区

这是整个界面的核心。

内容是：

- 差异文件列表
- 按 `新增` / `修改` / `删除` 标记
- 支持按类型筛选
- 显示总差异数量

### 3.3 右侧状态区

只展示当前目标仓库最关键的信息：

- 仓库名
- 当前分支
- 是否有未提交改动
- 未提交数量
- 未跟踪数量
- `提交`
- `推送`

## 5. 视觉重点

首屏视觉权重从高到低应当是：

1. 差异文件列表
2. 同步方向
3. 目标仓库状态
4. 同步按钮
5. 说明文字

不要让说明、提示、日志占据比差异列表更大的空间。

## 6. 关键交互

### 6.1 打开工具

- 自动恢复 A/B 路径
- 自动恢复上次方向
- 自动刷新差异列表
- 自动刷新目标仓库状态

### 6.2 切换方向

- 立即重新计算差异列表
- 立即切换右侧目标仓库状态

### 6.3 执行同步

- 成功后刷新差异列表
- 成功后刷新目标仓库状态

### 6.4 提交 / 推送

- 只针对当前目标仓库
- 不影响源仓库
